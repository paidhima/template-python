# Python Template: Copilot Language Profile

This profile extends the global baseline and applies to Python repositories.

Tooling defaults
- Prefer `ruff` for linting and formatting.
- Use `mypy` targets and strictness from repo configuration.
- Use `pytest` for tests.

Required checks before completion
1. Run lint checks for changed Python paths.
2. Run formatting checks for changed Python paths.
3. Run type checks when the repo uses mypy.
4. Run tests relevant to changed code.

Typical commands
- `ruff check .`
- `ruff format --check .`
- `mypy <targets from repo config>`
- `pytest`

Coding rules
- Add type hints for new or changed public APIs.
- Follow existing project structure and style.
- Do not add broad refactors outside task scope.
- Avoid adding `type: ignore` unless justified inline.

Documentation rules
- Update README or docs when behavior, setup, or interfaces change.
- Add docstrings/comments where they improve clarity for changed code.
- Do not mass-add docstrings/comments to untouched code.

Branch and PR policy
- Use `feature/*`, `fix/*`, or `refactor/*` branches for substantive changes.
- Treat the default branch (main/master) as protected unless user explicitly directs otherwise.

Version guidance
- Prefer modern Python syntax supported by the repository's minimum version.
- Call out compatibility or migration implications when versions are affected.
